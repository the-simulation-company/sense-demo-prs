# OCSF audit log: emit step_attempted_failed + step_skipped_prior_failure events

Spring Health and Ramp's auditors flagged that the OCSF audit log only records successfully-completed Playbook steps — failed steps and steps skipped because of a prior failure don't emit at all. Partial Playbook runs read in the log as shorter successful Playbooks, which is exactly the audit gap that loses SOC 2 examinations. This extends the OCSF schema with two event types: `step_attempted_failed` (step ran, error code/message/retry count captured) and `step_skipped_prior_failure` (downstream step that didn't run because of an upstream failure). Auditors now see the complete intended Playbook (CSL-2492).
