# Stop Okta seniority signal overriding request-content classifier

The auto-router was treating the Okta `seniority` org-context field as a routing signal with equal weight to the request-text classifier, so executive hardware and access requests were landing in the People Ops queue instead of IT. Two-line fix: demote seniority to a priority signal (it changes urgency tier within a queue, not which queue), and let any high-confidence (>0.85) IT/access content classifier match win against any org-context hint. Repros cleanly from Cursor, Webflow, and Synthesia's routing logs (CSL-2841).
