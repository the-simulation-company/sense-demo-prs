# Walk per-repo collaborators in GitHub deprovisioning step

Three customers (Cursor, Ramp, Flock Safety) reported that offboarded engineers retained per-repo write access on private repos where they were added as outside collaborators or via relaxed-membership policies. Today the deprov step walks org membership, team membership, and issues a final org-removal call — but doesn't walk per-repo collaborator entries. This extends the step to `GET /repos/{owner}/{repo}/collaborators` across every repo in the org and revoke per-repo entries for the offboarded user. Includes a one-time backfill scan against the offboarding audit log for existing customers (CSL-2779).
